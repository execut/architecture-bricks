# Требования к основному приложению:
1. Работает на базе данных PostgreSQL, развёрнутой в докере через конфиг docker-compose.yml. Схема БД:
   1. product
      1. id uuid
      2. name string
   1. product_event
      1. id uuid
      2. user_id uuid
      3. event string
      4. payload json
2. Приложение работает на языке go через библиотеку github.com/spf13/cobra
3. Основное приложение изолировано от его бизнес-логики через интерфейсы в файле contract.go
4. Структура приложения, example, его основная папка, содержит следующее:
   1. cmd/main.go - основное приложение
   2. cmd/commands - реализации команд
   3. contract/contract.go - интерфейсы с контрактом приложения
   4. app - различные реализации контрактов приложения
      1. v1_scenario_of_transaction - сценарий транзакции
         1. service.go - структура, которая реализует контракт приложения со всем функционалом по записи и чтению из БД

# Истории пользователя. Я как пользователь хочу:
1. Создавать товар с указанием uuid и названия
2. Получать параметры товара по uuid
3. Получать историю действий с товаром
4. Обновлять название товара
   1. Если название не изменилось, выдавать ошибку
5. Одобрять товар с указанием uuid модератора и причины одобрения "Moderator"
6. Отклонять товар с указанием uuid модератора и причины отклонения "Moderator"
7. Автоматически одобрять товар с названием Кот и Собака с указанием причины одобрения "Auto".
